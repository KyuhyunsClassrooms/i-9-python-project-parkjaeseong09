# 최종 보고서

## 1. 프로그램 주제와 목적

내 프로그램의 주제: 직원 실적 분석 및 성과급 계산기

이 프로그램을 만든 이유: 나의 진로와 관련이 있고 현실에서도 쓰일 것 같은 프로그램이라고 생각한다.


---

## 2. 문제 분해 및 모델링

### 입력
이름과 점수를 입력한다.

### 처리
점수에 따라 등급이 나뉘고 등급에 따라 성과급이 나뉜다.

### 출력
이름, 점수, 등급, 성과급을 출력한다.

---

## 3. 사용한 2차원 리스트 설명

2차원 리스트 이름: employee_data

한 행의 의미:  직원 1명의 전체 정보 묶음.

각 열의 의미: 0번 열은 직원의 이름, 1번 열은 등급 산출에 필요한 실적 점수이다.

| 열 번호 | 의미 | 실제 예시 |
|---:|---|---|
| 0번 열 |  |  |
| 1번 열 |  |  |
| 2번 열 |  |  |
| 3번 열 |  |  |

이 2차원 리스트를 사용한 이유:  만약 이름 리스트(['김철수', '이영희'])와 점수 리스트([95, 82])를 따로 관리했다면, 특정 직원을 삭제하거나 수정할 때 두 리스트의 순서가 어긋날 수 있다.따라서,  2차원 리스트는 이름과 점수를 하나의 세트로 묶어주기 때문에 데이터가 섞일 염려가 없다.

어떤 조건을 검사했는가?
직원의 실적 점수가 90점 이상인지, 80점 이상인지, 60점 이상인지를 검사하고 S,A,B,C 등급을 분류했다.
